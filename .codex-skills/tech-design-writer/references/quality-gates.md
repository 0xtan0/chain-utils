# Quality Gates

Run this checklist before final output.

## Hard Constraint Gates

- [ ] Output is Markdown.
- [ ] Output is divided into clear sections.
- [ ] No full implementation code is present.
- [ ] No tests or test cases are included.
- [ ] No unverified sources are cited.
- [ ] If charts are needed, charts are written in Mermaid.

## Content Completeness Gates

- [ ] Overview or problem statement exists.
- [ ] Goals and non-goals are explicit.
- [ ] Constraints and assumptions are explicit.
- [ ] High-level architecture is described.
- [ ] Interfaces and types are defined.
- [ ] Error and security model is defined.
- [ ] Public API surface is listed.
- [ ] Rollout and compatibility or migration is covered.
- [ ] Design considerations and tradeoffs are included.

## Design Quality Gates

- [ ] SOLID principles are reflected in boundaries and responsibilities.
- [ ] Security controls include validation, trust boundaries, and safe defaults.
- [ ] Naming and structure are consistent with the selected canonical style.
- [ ] Proposed design avoids unnecessary complexity.
